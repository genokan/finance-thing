# finance-thing v2 Rebuild — Build Plan

Implements `specs/2026-05-30-finance-thing-v2.md`. Sequenced so the data model lands first, then backend, then UI. Infra is deferred.

## Phase 1 — Data model (foundation)
1. Rewrite `schema.prisma`: add `Account`, `Holding`, hierarchical `Category` (bucket + monthlyBudget + appliesTo), `AppSetting`; extend `IncomeSource` (type, gross, payFrequency, tax config) + `IncomeDeduction`; extend `ExpenseItem` (kind, dueDate); restructure `Debt` (term + isZeroPromo + promo fields + kind + category); point `IncomeDistribution` and `PlaidItem` at `Account`; remove `InvestmentAccount`. Update snapshot children to reference `Account`/`Category`.
2. `prisma db push` + `prisma generate`.
3. Update seed: default category tree (with buckets), admin user.

## Phase 2 — Backend
4. `accounts` route: CRUD + nested holdings CRUD; value computation helper.
5. `categories` route: hierarchical CRUD (parent/child), budget targets.
6. `budgets` route/aggregation: actual vs target per category + 50/30/20 roll-up.
7. `income` route + `tax` service (`lib/tax/2026.ts`): FLAT and BRACKET estimators, FICA, state; net take-home breakdown.
8. `expenses` route: recurring vs one-time handling.
9. `debts` route: new fields, promo alerts off `promoEndsAt`.
10. `plaid` service: sync balances into `Account`.
11. `import` service: column-mapping CSV → accounts/holdings.
12. `settings`: user settings + admin `AppSetting` CRUD (admin-gated).
13. `snapshots`: live computation everywhere; `node-cron` monthly auto-snapshot; keep manual trigger.
14. `dashboard` + `insights`: recompute over new model.

## Phase 3 — Frontend
15. Accounts + Holdings (replaces Investments add flow).
16. Budgets page (50/30/20 + editable category tree with targets).
17. Income with tax breakdown + FLAT/BRACKET toggle.
18. Debt updates (term, 0% promo, kind, category).
19. Expenses recurring/one-time.
20. Settings split + admin app-settings UI.
21. Import column-mapping UI.
22. Dashboard charts + insights deep-dive.

## Phase 4 — Deferred infra
23. Vault secret loading; 24. CI → GHCR; 25. Tests (unit/regression/Playwright); 26. Security (scanning, Dependabot, pentest, data review).
